# Team Pack Walkthrough Example

This redacted example shows how a public-safe team pack is expected to behave.

1. A team baseline declares shared policies and playbooks.
2. Role profiles add scoped guidance for engineer, product, and research work.
3. Shared adapter text remains canonical source material, not a live runtime overwrite.
4. A preview report checks referenced files and public-safe classification.
5. Future apply behavior must generate review candidates before touching private adapters.

Example pack: `sample-assets/team-pack/team-pack.yaml`
